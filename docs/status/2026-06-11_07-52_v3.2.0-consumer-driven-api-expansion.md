## E) WHAT WE SHOULD IMPROVE

### Critical (blocks CI)

1. **Coverage gap in `scan.go`** — `ScanProject` has filesystem walking paths that are hard to test with `fstest.MapFS` (directory traversal). The `collectGoFiles` function uses `fs.WalkDir` which requires real directory structure. Consider adding integration tests with `os.DirFS(t.TempDir())` setups, or use `t.Skip` in CI with a lower threshold for the new file.

2. **CI coverage threshold** — `ci.yml` has `98%` threshold. Current is `96.9%`. Either:
   - Add more tests to cover `scan.go` edge cases
   - Lower threshold temporarily
   - Exclude `scan.go` from coverage counting (not recommended)

### High Priority

3. **`bdd_extended_test.go` missing new generator tests** — The BDD extended tests should have DescribeTable entries for the 7 new generators, testing filename patterns, content markers, and edge cases.

4. **`scan.go` goconst warnings** — The `generatorExclusionReasons` map now uses constants, but the `exclusionsForGenerator` switch still uses `string(FilterSQLC)` etc. in case labels which is not ideal. Consider refactoring to use `FilterReason` directly.

5. **Example test counts are fragile** — `ExampleAllFilterOptions` and `ExampleAllFilterReasons` hard-code output counts (19, 21). Every new detector will break these. Consider deriving the expected count programmatically.

### Medium Priority

6. **Test file goconst noise** — 14 goconst warnings in test files. Consider extracting common test strings to `testdata_test.go` constants.

7. **`filter_content_test.go` golines formatting** — LSP reports formatting issues. Run `golangci-lint fmt` again.

8. **No table-driven tests for new `Is*Generated` functions** — The 7 new `Is*` functions are tested implicitly through integration tests but lack dedicated table-driven unit tests.

9. **`ExclusionPattern` returns regex strings** — The patterns are returned as raw strings but aren't validated as compilable regexes. Add a test that `regexp.Compile` succeeds for all patterns.

10. **Website docs don't cover `ScanProject` API surface** — No dedicated guide page for the project scanner. Only the changelog mentions it.
