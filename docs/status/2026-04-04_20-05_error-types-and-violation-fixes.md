## C) NOT STARTED

### MEDIUM Violations — `os.Stat` with blank identifier

Two locations use `if _, err := os.Stat(path); err == nil`:

1. `project.go:23` — checking if marker file exists
2. `sqlc.go:118` — checking if config file exists

**Verdict**: These are **idiomatic Go** for file-existence checks. The blank identifier correctly discards
the `FileInfo` that isn't needed. The `err` IS checked (via `err == nil`). These are false positives
from the analyzer — **no fix needed**.

## G) TOP #1 QUESTION

**Should `GetSQLOutputDirs` return partial results on config parse failure, or fail fast?**

The fix I applied makes it fail fast (return error immediately). However, the original behavior was
to log a warning and continue — this could be useful when scanning projects with multiple independent
sqlc configs where one broken config shouldn't block the others. Should this be configurable via an
option like `WithContinueOnError()`?
