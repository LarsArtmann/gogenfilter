## Self-Review: What Could Have Been Better

### What I forgot / missed

1. **`testhelpers` package duplication** — I created `sqlcDBContent` in `example_test.go` when `testhelpers.SQLCGeneratedContentTest` already exists. Correct decision (examples should be self-contained for godoc), but I should have noted this explicitly.

2. **AGENT.md claimed DOMAIN_LANGUAGE.md was stale** — The status report said "May not reflect FilterDetailed, trace support, or dependents page" but all three ARE already documented. I verified this during research but didn't correct the AGENTS.md entry.

3. **Website CI path filter analysis was incomplete** — I correctly noted `website/**` covers `dependents.astro`, but the deeper finding is that `dependents.astro` uses `GITHUB_TOKEN` which needs `actions: read` permission. The current `website.yml` only has `contents: read` — this may need verification.

### What I could still improve

4. **Accessibility fixes are small and high-impact** — The audit found 3 specific a11y issues (decorative dots missing `aria-hidden`, dependents table missing `<caption>`, star header missing `aria-label`). These are 2-minute fixes each.

5. **DOMAIN_LANGUAGE.md is missing 9 exported symbols** — `DetectReasonReader`, `FilterPathsDetailed`, 11 `Is*Generated` functions, `AllFilterOptions`/`AllGeneratorOptions`/`AllFilterReasons`, `MatchPattern`, `FindSQLCConfigsFS`/`GetSQLOutputDirsFS`, `WithFS`, `ErrorCoder`, sentinel errors. Should be added for completeness.

6. **`SQLCConfigError.Operation` uses bare strings** — Magic strings like `"find"`, `"walk"`, `"read"`, `"parse"`, `"collect-output-dirs"` should be typed `SQLCOperation` constants.

7. **`IsValid()` allocates on every call** — `slices.Contains(AllFilterOptions(), o)` creates a new slice each time. A package-level `validOptions` map would be more efficient for a library API.

## Architecture Observations

### Type Safety Opportunities

| Current | Proposed | Benefit |
|---------|----------|---------|
| `SQLCConfigError.Operation string` | `SQLCConfigError.Operation SQLCOperation` | Compile-time checking, IDE completion |
| `IsValid()` → `slices.Contains(AllFilterOptions(), o)` | Package-level `validOptions` map | Zero allocation on hot path |
| Raw `map[FilterOption]struct{}` in 8+ places | `type optionSet` with helper methods | Cleaner encapsulation |
| `FilterPaths() ([]bool, error)` | Deprecate in favor of `FilterPathsDetailed()` | Eliminate weakest API |

### No Split Brains Found

The `detectors` table remains the single source of truth. All derived functions (`AllFilterOptions`, `AllGeneratorOptions`, `AllFilterReasons`, `filterOptionToReason`) derive from it. This is excellent architecture.

### Dependencies Are Optimal

Neither `doublestar/v4` nor `go-faster/yaml` can be replaced with Go 1.26 stdlib. All 4 deps are justified.
