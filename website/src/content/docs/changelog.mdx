## [Unreleased]

### Added

- `DetectReasonReader(filePath string, r io.Reader, opts ...FilterOption) (FilterReason, error)` — detection from an `io.Reader`
- `FilterStats.FilteredFiles(reason FilterReason) []string` — returns file paths filtered for a given reason (defensive copy, safe to mutate)
- `Filter(filePath string) (bool, error)` — replaces `ShouldFilter` with cleaner name; `MustFilter` removed
- SQLC config v1 format test coverage
- Cross-platform path matching tests
- `errorCodeDefs` single-source-of-truth table
- `map[FilterOption]struct{}` replaces `map[FilterOption]bool`
- `fmt.Stringer` on all 5 phantom types
- Runnable examples for `Filter`, `WithFS`, `WithIncludePatterns`, `GetStats`, and `DetectReasonReader`
- Error handling examples (`errors.Is`, `ErrorCode()`, `Help()`, `CodeHelp`, `AllErrorCodes`, `FindProjectRoot`)
- CI benchmark step (`go test -bench=. -benchmem`)

### Changed

- **`ShouldFilter` renamed to `Filter`** — the method `ShouldFilter(filePath string) (bool, error)` is now `Filter(filePath string) (bool, error)`. The `MustFilter` panic-on-error variant has been removed; callers should handle errors explicitly.
- **`Enabled()` and `Disabled()` removed** — filter is enabled by passing options to `NewFilter()`; `NewFilter()` with no args is disabled. This eliminates silent misconfiguration.
- **`IsValid()` methods derived from tables** — `FilterOption.IsValid()` and `FilterReason.IsValid()` iterate `AllFilterOptions()`/`AllFilterReasons()` instead of manual switches
- **Error codes derived from `errorCodeDefs` table** — `AllErrorCodes()` and `CodeHelp()` from one table
- **`filteredByReason` map unexported** — use `FilteredBy(reason)` accessor for per-reason breakdown
- **Phantom types no longer use format strings** — `String()` methods simplified
- **SQLC patterns consolidated** — `sqlcFilePatterns`/`sqlcCodePatterns` inlined into consuming functions
- **`WithFilterOptions` reuses `optionsMap`** — `FilterAll` expansion no longer duplicated between `filter.go` and `detection.go`
- **`filteredFiles` moved to `MetricsMixin`** — file path tracking via `FilteredFiles()` accessor
- **`slog` dependency removed** — `warnMultipleSQLCConfigs` removed entirely
- **Phantom types used directly** — eliminated 8+ explicit `string()` casts via `fmt.Stringer`
- **`sqlcConfigError` bridge removed** — internal callers use `newSQLCConfigError` with typed phantom values
- **`Validatable` interface unexported** — renamed to `validatable`; internal generic constraint only
- **`IsEnabled()` derives state from configuration** — `len(f.options) > 0 || len(f.includePatterns) > 0 || len(f.excludePatterns) > 0`
- **`enabled bool` field removed from `Filter` struct** — state is implicit, not stored

### Fixed

- **Silent misconfiguration** — `NewFilter(WithFilterOptions(FilterAll))` (without `Enabled()`) now works; passing options automatically enables the filter
- **Data race in `Metrics.filteredFiles`** — field unexported; was accessible without mutex protection
- **Leaky `fs.FS` abstraction** — `detectReasonFS` no longer falls back to `os.ReadFile` for custom filesystems
- **README metrics example** — `TotalFilesChecked == 3` (was incorrectly `1`)

### Removed

- **`os.ReadFile` fallback in `detectReasonFS`** — custom `fs.FS` implementations now behave correctly
- `warnMultipleSQLCConfigs` function and all `slog` usage
- `sqlcConfigError()` bridge function
- `Causable` interface (was unused)

## [0.1.0] — 2026-04-04

### Added

- Initial release
- Two-phase detection: filename-based (zero I/O) then content-based
- 11 generator detectors: sqlc, templ, go-enum, protobuf, oapi-codegen, deepcopy-gen, wire, moq, mockgen, stringer, generic
- Functional options API: `WithFilterOptions()`, `WithFS()`, `WithIncludePatterns()`, `WithExcludePatterns()`
- `fs.FS` abstraction for testing
- Thread-safe metrics
- SQLC config discovery
- Glob pattern matching with `**` support
- Branded error types with sentinel errors
- Phantom types for type-safe API boundaries
