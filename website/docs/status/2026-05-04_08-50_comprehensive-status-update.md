1. **Remove dead `Section.astro` component** — It's defined but never imported. Either integrate it into `index.astro` (replace all 6 `<div data-animate>` wrappers with `<Section>`) or delete it. Currently it creates confusion about intended architecture.

2. **Remove dead `CodeBlock.astro` component** — Never imported. HeroSection has its own inline code block. Either use `CodeBlock` in HeroSection or delete the component.

3. **Type the `icons` record keys** — `Record<string, string>` in `features.ts` should be `Record<Feature['icon'], string>` to catch typos at compile time.

4. **Add `focus-visible` styles** — Keyboard users need visible focus indicators:
   ```css
   :focus-visible {
     outline: 2px solid var(--color-accent);
     outline-offset: 2px;
   }
   ```

5. **Add canonical URL meta tag** to `LandingLayout.astro`:
   ```html
   <link rel="canonical" href={Astro.url.href} />
   ```

### Short-term (15-30 min each)

6. **Integrate `Section.astro` into `index.astro`** — Replace 6 duplicated `<div data-animate class="py-24 relative">` wrappers with `<Section>` component. This makes the section spacing a single source of truth.

7. **Use `CodeBlock.astro` in HeroSection** — Replace the inline code block in HeroSection with the `CodeBlock` component for consistency. Currently HeroSection has its own custom code block with macOS window chrome (the window chrome is intentional, so maybe keep it separate).

8. **Run Lighthouse audit** — `npx lighthouse https://gogenfilter.lars.software --view` to get performance, accessibility, SEO, and best practices scores. Expected issues: render-blocking fonts, missing `<html lang="en">`, missing viewport meta on docs pages.

9. **Browser visual QA** — Open site at 375px (iPhone), 768px (iPad), 1440px (desktop). Verify:
   - Mobile nav toggle works
   - All generator cards are tappable
   - Scroll animations trigger correctly
   - Copy button appears on code hover
   - OG images render in Slack/Discord link previews

10. **Add HTML validation to CI** — `html-validate` as a separate job in `ci.yml` to catch malformed HTML before deployment.

11. **Verify OG image generation** — The `/og/docs/api/filter.png` etc. files are generated but have not been visually verified. Check that text renders correctly and logos appear.

### Medium-term (30-60 min each)

12. **JSON-LD structured data** — Add `@type: Package` schema to the landing page for Go library rich results in Google.

13. **Source real brand logos** — The 11 generator logos are custom SVGs matching the site's dark theme. This is actually a *feature* (consistency), not a bug. But sqlc, protobuf, and templ have official logos that could be used if properly licensed.

14. **Add `aria-current="page"` to active nav link** — The docs nav should mark the current page as active for screen readers.

15. **Improve `generators.ts` data quality** — Some URLs are personal repos (`go-enum`, `moq`), some are official (`sqlc.dev`, `templ.guide`). Consider adding a `verified` or `official` field to distinguish them.
